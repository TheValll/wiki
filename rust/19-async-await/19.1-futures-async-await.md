# 19.1 вҖ” Futures, `async`, `.await`

A **future** is a value that represents *work in progress* вҖ” a computation that may not have produced its result yet. In Rust, the `Future` trait is the foundation:

```rust
pub trait Future {
    type Output;
    fn poll(self: Pin<&mut Self>, cx: &mut Context) -> Poll<Self::Output>;
}

pub enum Poll<T> {
    Ready(T),
    Pending,
}
```

Calling `poll` on a future is asking *"are you done yet?"* вҖ” it returns `Ready(value)` if so, or `Pending` if not. You almost never write `poll` by hand. Instead, you write `async` blocks, and the compiler builds the future for you.

## 19.1.1 вҖ” `async fn` returns a future

```rust
async fn fetch_user(id: u64) -> User {
    let row = db.query(id).await;
    User::from_row(row)
}
```

This function does **not** return a `User`. It returns a value of type `impl Future<Output = User>`. Calling `fetch_user(42)` is **lazy** вҖ” nothing happens. The body runs only when something polls the future.

```rust
let fut = fetch_user(42);    // nothing happens вҖ” `fut` is a paused computation
let user = fut.await;        // now the work runs, until completion
```

This laziness is the single most important thing to remember: **futures don't run themselves**.

## 19.1.2 вҖ” `.await`: yielding control

Inside an async context, `expr.await` says *"poll this future. If it's `Ready`, continue. If it's `Pending`, hand control back to the executor вҖ” wake me up when there's progress to make."*

```rust
async fn do_work() {
    let a = step_one().await;        // wait for step_one
    let b = step_two(a).await;       // then step_two
    println!("{}", b);
}
```

Visually, an `async fn` is **a function that pauses at every `.await`**. The compiler rewrites the body into a state machine: each `.await` becomes a saved checkpoint where execution can suspend and later resume.

## 19.1.3 вҖ” `.await` only works inside `async`

`.await` is parsed only inside an `async fn`, an `async {}` block, or an `async move {}` block. Outside, you must hand the future to an executor (В§19.2):

```rust
fn main() {
    let fut = fetch_user(42);
    let user = block_on(fut);   // synchronous wait, runs the runtime here
}
```

## 19.1.4 вҖ” Anatomy of `async`

| Form | What it produces |
|------|------------------|
| `async fn foo() -> T` | A function that returns `impl Future<Output = T>` |
| `async { вҖҰ }` | An expression of type `impl Future<Output = вҖҰ>` |
| `async move { вҖҰ }` | Same, but the block takes ownership of captured variables |
| `async \|x\| { вҖҰ }` | An async closure (stable as of recent Rust versions) |
