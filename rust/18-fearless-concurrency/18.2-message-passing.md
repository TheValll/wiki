# 18.2 — Message passing with channels (`mpsc`)

> *"Do not communicate by sharing memory; instead, share memory by communicating."* — Go proverb, embraced by Rust.

A **channel** is a one-way pipe between threads: one side `send`s values, the other side `recv`s them. Ownership of each value transfers across the channel — once sent, the producer can no longer use it.

`mpsc` = **multiple producers, single consumer**.

```rust
use std::sync::mpsc;
use std::thread;

fn main() {
    let (tx, rx) = mpsc::channel();   // (transmitter, receiver)

    thread::spawn(move || {
        let val = String::from("hello");
        tx.send(val).unwrap();
        // println!("{}", val);  // ERROR: val was moved into send()
    });

    let received = rx.recv().unwrap();
    println!("Got: {}", received);    // Got: hello
}
```

`tx.send(val)` takes ownership of `val`. The compiler then refuses any later use — no risk that the sender keeps reading while the receiver mutates.

## 18.2.1 — `recv` vs `try_recv`

| Method | Behavior |
|--------|----------|
| `rx.recv()` | **Blocks** until a value arrives or all senders have been dropped |
| `rx.try_recv()` | **Returns immediately** — `Ok(val)` if available, `Err` otherwise |

Use `recv()` when the thread has nothing else to do. Use `try_recv()` in a loop when you want to interleave channel polling with other work.

## 18.2.2 — Receiver as an iterator

`rx` implements `Iterator`: `for val in rx` blocks for each value and exits cleanly when the channel closes (i.e., all `tx` clones are dropped).

```rust
let (tx, rx) = mpsc::channel();

thread::spawn(move || {
    for s in ["one", "two", "three"] {
        tx.send(s.to_string()).unwrap();
        thread::sleep(Duration::from_millis(200));
    }
});

for received in rx {
    println!("Got: {}", received);
}
```

## 18.2.3 — Multiple producers via `tx.clone()`

The `tx` end is **clonable**. Each clone is an independent sender feeding the same receiver.

```rust
let (tx, rx) = mpsc::channel();
let tx2 = tx.clone();

thread::spawn(move || tx.send(String::from("from A")).unwrap());
thread::spawn(move || tx2.send(String::from("from B")).unwrap());

for msg in rx {
    println!("{}", msg);   // both arrive, order non-deterministic
}
```

## 18.2.4 — Sync vs async channels

| Constructor | Capacity | Behavior on `send` when buffer full |
|-------------|----------|-------------------------------------|
| `mpsc::channel()` | **Unbounded** | Always returns immediately |
| `mpsc::sync_channel(n)` | Bounded to `n` | **Blocks** until there is room |

`sync_channel(0)` is a *rendezvous* channel — every `send` blocks until a `recv` happens, no buffering at all.
