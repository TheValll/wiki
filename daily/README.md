# Daily — Learning Journal

Per-day log of what was worked on, what is worth retaining, and gaps to revisit. Built around the **20-min end-of-day ritual** (cf. [`how-i-learn.md`](../how-i-learn.md)).

**Language: English** — the user writes daily entries in English as a deliberate language-practice exercise. The agent follows the same convention when pre-filling or scaffolding entries. Chat conversation stays in French as usual; only the journal files are English.

---

## Convention

- **One file per day:** `YYYY-MM-DD.md`
- **Flat folder** for now; if it grows past ~60 entries, regroup by month (`2026-05/`, `2026-06/`, …).
- **No RECAP file** — a journal is a chronological log, not a curriculum.
- **Numbering is implicit** (the date) — no `X.Y-` prefix.

---

## Daily template

Each file has 3 sections:

| Section | Purpose | Time |
|---|---|---|
| **Tracks (raw log)** | 1 line per active learning track (Hiari, MML, Kata, DeepSight, Paper) — what got worked on | 5 min |
| **Articulation of the day** | THE concept of the day, re-explained from memory in 3-5 sentences. Compare with the source. Note the delta. | 10 min |
| **Questions / gaps** | What did not click today — to revisit | 5 min |

The articulation step is the **core**: the only one that produces real long-term retention. Log + questions are scaffolding.

> Anki cards are written by Valentin himself, not generated by the agent (generation effect = better retention). The agent does not maintain an "Anki candidates" section in entries.
