# Part 2.2 вҖ” SoC (System on Chip)

The term **SoC** is used loosely in the embedded world. Sometimes it means a microcontroller. Sometimes a beefed-up microcontroller. Sometimes "anything with multiple subsystems on one die". This page clarifies which is which вҖ” and how the book uses the term.

---

## 2.2.1 вҖ” The framing

| Manufacturer style | What they call an "SoC" |
|---|---|
| **MCU manufacturers** (Espressif, ST, Nordic) | Their MCUs вҖ” same chip, marketed as SoC |
| **General-purpose chip makers** (Intel, AMD, Qualcomm) | An MPU + chipset bundled in one package (no main memory) |

**This book and wiki use "SoC" and "MCU" interchangeably**, following the Espressif convention (the ESP32 family is officially "SoCs"). When you see "ESP32 SoC" in a datasheet, mentally translate to "ESP32 MCU".

---

## 2.2.2 вҖ” What an SoC technically is

The cleanest definition: an SoC is a **beefed-up microcontroller**. Same general layout as an MCU (memory + peripherals + cores integrated on one die), but with:

- **Higher performance** (faster cores, higher clock frequency)
- **Multiple cores** sometimes (vs typically one for a "pure" MCU)
- **Optional accelerators** вҖ” GPU for graphics, FPGA fabric for custom logic, neural network engines, video encoders, etc.

The shape mirrors an MCU. The difference is power and optional acceleration.

---

## 2.2.3 вҖ” Properties at a glance

| Axis | MPU | MCU | SoC |
|---|---|---|---|
| **Cores** | 1+ | 1 | 1+ |
| **Performance** | High | Low | Medium вҶ’ High |
| **Power consumption** | High | Low | Medium вҶ’ High |
| **Memory** | External | Integrated | Integrated |
| **Peripherals** | External | Integrated | Integrated |
| **Acceleration units** | External | None | Integrated (optional) |

The ESP32 family fits cleanly in the SoC column: integrated memory + peripherals (like an MCU), but with optional dual cores and Wi-Fi/BLE accelerators (more powerful than a "pure" MCU).

---

**Mental shortcut:** if marketing says "SoC", treat it as an MCU. If it has a GPU or FPGA fabric, treat it as an MCU+ (which is also called an SoC).
