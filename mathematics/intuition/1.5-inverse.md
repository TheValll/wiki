# 1.5 — The Matrix Inverse: The Un-Machine

Pure-intuition companion to §1.5 of [`../01-linear-algebra.md`](../01-linear-algebra.md). No formulas, no exercises.

---

The inverse `A⁻¹` is the machine that **undoes** machine A. If A stretched, rotated, or sheared an arrow `v` into `A·v`, then `A⁻¹` takes `A·v` and brings it back to exactly `v`. Run A, then run `A⁻¹` — the sheet returns to where it started.

**The physical image — running the machine in reverse.** Same rubber sheet, same conveyor belt as §1.4. Machine A deforms the sheet. Place a second machine right after A whose job is to **precisely cancel** every stretch and every shear A applied. The sheet exits in its original shape, as if nothing had happened. That second machine is `A⁻¹`.

Because A and `A⁻¹` cancel each other, their combined action is **the identity machine** — the machine that does nothing, the machine that leaves every arrow exactly where it started. In matrix language that machine is `I`, the identity. So `A · A⁻¹ = I` is just the sentence "deform, then un-deform = no change", written as a matrix equation.

---

### When the inverse does not exist

Not every machine has an un-machine. The rule is sharp:

**A machine is reversible if and only if it does not destroy any dimension.**

Why? Look back at §1.6. If `det A = 0`, the machine squashed the sheet flat onto a lower-dimensional shape. Many different input arrows collapsed onto the same output arrow. There is no way to ask "which of those inputs did you come from?" — the information has been lost. Un-squashing a pancake back into a square is not something even a more clever machine can do.

So the dividing line is:

- **`det A ≠ 0`** → the machine preserved all dimensions → the inverse exists → reversible.
- **`det A = 0`** → a dimension was crushed → the inverse does **not** exist → "singular matrix".

This is the same fact from §1.6, seen from the inverse's perspective. "No lost information" and "the un-machine exists" are two names for the same truth.

---

### What makes an inverse tick

Mechanically, if A stretches the x-axis by 3 and rotates by 45°, then `A⁻¹` has to **shrink the x-axis by 3** and **rotate by −45°** — apply the opposite of every transformation, in the reverse order.

Two clean consequences:

- **`(A · B)⁻¹ = B⁻¹ · A⁻¹`.** To undo "B first, then A", undo A first, then undo B. The order reverses because you are running the tape backwards — the last step in is the first step out. Like socks-then-shoes on the way in: shoes-then-socks on the way out.
- **`(A⁻¹)⁻¹ = A`.** Undoing the undoer brings you back to the original machine. Pressing rewind twice = never pressing it.

---

**What to remember.**

The inverse is the **un-machine**. It exists exactly when A has not destroyed any dimension (`det A ≠ 0`), because destroyed information cannot be recovered. `A · A⁻¹ = I` means "deform, then un-deform = do nothing". Every time you solve a system `A·x = b` by writing `x = A⁻¹·b`, you are saying "b is the image of some unknown x — run the un-machine on b to find out where x came from".
