# 1.3 — Matrix × Vector: A Recipe for Mixing Columns

Pure-intuition companion to §1.3 of [`../01-linear-algebra.md`](../01-linear-algebra.md). No formulas, no exercises.

---

A matrix is not an abstract grid of numbers — it is a **list of columns**, and each column is itself a vector, an arrow in space. The input vector `v` is a **recipe**: each of its coordinates says "take this much of column 1, this much of column 2..." and you add them all together. The result `A·v` is that mixture, nothing more.

**The physical image — mixing paints.** A painter has several jars of pure pigment. Each **column of the matrix is one jar**. The vector `v` is the **dosage card**: "3 parts of this jar, 2 parts of that one, 0 of the third one". You pour, you stir, you get a final color. Change the recipe (new `v`), same jars, different color. Change the jars (new matrix), same recipe, also different color.

This image makes several things obvious at a glance:
- why the operation is *linear* — doubling the recipe doubles the final color,
- why the number of columns of `A` must match the number of components of `v` — a three-dose recipe assumes exactly three jars.

---

### Step-by-step construction

Take a 2×2 matrix whose two columns are arrows in the plane. Take `v = (3, 2)` as the recipe.

**Step 1.** Draw the two columns of `A`, `col₁` and `col₂`, starting from the origin.
```
          col₂
           ↗
          ╱
         ╱
        •────────→ col₁
```

**Step 2.** The recipe says "3 of col₁". Stretch `col₁` by a factor of 3.
```
        •════════════════════→  3·col₁
```

**Step 3.** Add "2 of col₂", tip-to-tail (parallelogram rule).
```
                              ↗ 2·col₂
                            ╱
                          ╱
        •═════════════════→
                  3·col₁
```

**Step 4.** The final tip *is* `A·v`.
```
                            • ← A·v
                          ╱
                        ╱
        •─────────────→
```

---

**What this makes obvious.** `A·v` can only reach points accessible by mixing the columns. If the two columns happen to be parallel (redundant), every possible result is stuck on a single line — the matrix has lost a dimension. That is the seed of **rank**, of **zero determinant**, and of **non-invertible matrices**: ideas that will return in later sections.

---

**What to remember.**

Matrix × vector = **a recipe for mixing the matrix's columns**. The columns are arrows, `v` tells you how much of each to take, and you add them together. Every rotation, projection, system of equations, or neural-network layer is a variant of this same operation: mix the columns with a recipe.
