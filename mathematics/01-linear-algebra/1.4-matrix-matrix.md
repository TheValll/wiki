# 1.4 — Matrix × Matrix Multiplication

**What it does:**
Composes two linear transformations. Multiplying `B` then `A` is the same as applying a single combined matrix `AB`. Essential whenever you chain operations (rotate, then translate, then scale).

Imagine two **assembly lines** in sequence. The first line (matrix B) turns raw ingredients into subassemblies. The second line (matrix A) turns subassemblies into finished products. Instead of running both lines every time, you can build a **super-machine** `AB` that does both steps in one pass. Matrix multiplication *builds* that super-machine.

## Intuition — Two Machines in a Row, Collapsed Into One

Multiplying two matrices is **chaining two transformations into a single transformation**. The result `AB` is a new matrix that, applied to any vector, does in one step what `B` then `A` would do in two steps.

**The physical image — two shaping machines in sequence.** Picture a rubber sheet travelling on a conveyor belt. It first enters **machine B**, which stretches and shears it one way. The already-deformed sheet then enters **machine A**, which reshapes it a second time. The sheet comes out twice-shaped. Instead of running both machines, an engineer could design a **super-machine `AB`** that produces the exact same final shape in a single pass. Matrix multiplication is the mathematical act of *designing that super-machine*.

Two consequences fall out of this image immediately:

- **Order matters.** Machine A first then machine B generally leaves the sheet in a different shape than the other way around. That is why matrix multiplication is **non-commutative**: in general `AB ≠ BA`. Swap the machines, swap the result — there is nothing mysterious about it.
- **The notation looks reversed on purpose.** We write `AB` to mean "B first, then A", because we read `A(B·v)` right-to-left — exactly the order in which the sheet meets the machines. The matrix sitting closest to the vector is the machine the vector enters first.

---

### Step-by-step construction

We follow the unit square — the undeformed sheet — through a cascade of two concrete machines, watching what happens to the two basis arrows `e₁` and `e₂`.

- **Machine B = horizontal stretch by 2.** Sends `e₁ = (1, 0)` to `(2, 0)`; leaves `e₂ = (0, 1)` untouched.
- **Machine A = 90° counter-clockwise rotation.** Sends any horizontal arrow upward, any vertical arrow leftward.

**Step 1.** The pristine sheet. The two basis arrows sit at `(1, 0)` and `(0, 1)`.
```
     ↑ e₂  at (0, 1)
     │
     │
     •───→ e₁  at (1, 0)
```

**Step 2.** The sheet enters **machine B**. `e₁` is stretched to `(2, 0)`, `e₂` does not move. The square is now a **horizontal 2×1 rectangle**.
```
     ↑ col₂ of B  at (0, 1)   (unchanged)
     │
     │
     •──────────→ col₁ of B  at (2, 0)   (twice as long)
```

**Step 3.** The rectangle enters **machine A**. Everything rotates 90° counter-clockwise. The long arrow at `(2, 0)` swings up to `(0, 2)`; the short arrow at `(0, 1)` swings left to `(−1, 0)`. The rectangle **tips over** into the upper-left quadrant.
```
                   ↑  at (0, 2)   ← was col₁ of B, now rotated up
                   │
                   │
                   │
       ←───────────•  at (−1, 0)   ← was col₂ of B, now rotated left
```

**Step 4.** The super-machine `AB` performs both steps in a single pass. Its columns are exactly those two final positions. Starting from the original unit square, we land **directly** on the tipped-over rectangle without visiting the intermediate shape.
```
     Original unit square  ───── AB (single pass) ─────→

                   ↑ col₁ of AB  at (0, 2)
                   │
                   │
                   │
       ←───────────•  col₂ of AB  at (−1, 0)
```

---

**What this makes obvious.**

- **The hand-computation rule is geometry, not a trick.** Each column of `AB` is "where does that basis arrow end up after both machines" — which is machine A applied to the matching column of B. The arithmetic you crunch with a pencil is just this sentence, turned into numbers.
- **Non-commutativity is physical.** Swap the order: rotate first, then stretch horizontally. The basis arrows end up somewhere different — the final shape is different. Of course the super-machine is different too. Nothing algebraic to memorize.
- **A cascade of linear machines is still a linear machine.** Each machine keeps straight lines straight and leaves the origin fixed. Two of them in sequence do too. That is why the combined super-machine is still a matrix, not some exotic new object — and why robot kinematic chains, graphics pipelines, and stacks of linear network layers can each be described by a single equivalent matrix.

**What to remember.** Matrix × matrix = **two sequential machines, pre-collapsed into one**. Each column of `AB` answers "where does this basis arrow land after passing through both machines, B then A?". Order matters because the machines act in sequence, and different sequences sculpt the sheet differently. This single operation is what runs robot kinematic chains (base → shoulder → elbow → wrist), graphics pipelines (model × view × projection), and every deep network that stacks linear layers.

## Formula

For `A` of shape (m, n) and `B` of shape (n, p):
```
(AB)ᵢⱼ = Σₖ Aᵢₖ · Bₖⱼ
```

## Simple example (2×2 · 2×2)

```
A = [1  2]    B = [5  6]
    [3  4]        [7  8]

AB = [1·5 + 2·7   1·6 + 2·8]   = [19  22]
     [3·5 + 4·7   3·6 + 4·8]     [43  50]
```

## Complex example (3×3 · 3×3)

```
A = [1  0  2]    B = [2  1  0]
    [0  3  1]        [1  0  1]
    [4  1  0]        [0  2  3]

AB[0,0] = 1·2 + 0·1 + 2·0 = 2
AB[0,1] = 1·1 + 0·0 + 2·2 = 5
AB[0,2] = 1·0 + 0·1 + 2·3 = 6
AB[1,0] = 0·2 + 3·1 + 1·0 = 3
AB[1,1] = 0·1 + 3·0 + 1·2 = 2
AB[1,2] = 0·0 + 3·1 + 1·3 = 6
AB[2,0] = 4·2 + 1·1 + 0·0 = 9
AB[2,1] = 4·1 + 1·0 + 0·2 = 4
AB[2,2] = 4·0 + 1·1 + 0·3 = 1

AB = [2  5  6]
     [3  2  6]
     [9  4  1]
```
→ Used in URDF chains: base_link → shoulder → elbow → wrist collapsed into a single transform.
