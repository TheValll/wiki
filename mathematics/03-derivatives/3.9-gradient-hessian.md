# 3.9 — Gradient and Hessian

**What it does:**
Packages all partial derivatives of a multivariable function into structured objects.
- The **gradient** `∇f` is the vector of all first partial derivatives — it points toward the steepest ascent of `f`.
- The **Hessian** `H` is the matrix of all second partial derivatives — it describes local curvature (how the landscape bends).

Think of `∇f` as the **compass needle** on the topographic map: it always points uphill, and its length tells you how steep the climb is. Think of `H` as a **curvature sensor**: at a valley floor it reads "bowl" (positive curvature everywhere), at a ridge "dome" (negative curvature), at a pass "bowl in one direction, dome in another" (saddle).

**Formulas (for `f : ℝⁿ → ℝ`):**

Gradient (n × 1 column vector):
```
∇f = [ ∂f/∂x₁ ]
     [ ∂f/∂x₂ ]
     [   ...  ]
     [ ∂f/∂xₙ ]
```

Hessian (n × n symmetric matrix — by Schwarz's theorem `f_xy = f_yx` for smooth `f`):
```
H = [ f_xx   f_xy   ...   f_xn ]
    [ f_yx   f_yy   ...   f_yn ]
    [  ...    ...   ...    ... ]
    [ f_nx   f_ny   ...   f_nn ]
```

For `f(x, y)` (2 variables):
```
∇f = [ f_x ]        H = [ f_xx   f_xy ]
     [ f_y ]            [ f_yx   f_yy ]
```

**Simple example (2D):**
```
f(x, y) = x² + 3xy + 2y²

f_x = 2x + 3y                f_xx = 2     f_xy = 3
f_y = 3x + 4y                f_yy = 4     f_yx = 3

∇f = [ 2x + 3y ]        H = [ 2   3 ]
     [ 3x + 4y ]            [ 3   4 ]
```

**Complex example (Rosenbrock "banana" function — classic optimization benchmark):**
```
f(x, y) = (1 − x)² + 100·(y − x²)²

f_x = −2·(1 − x) + 100·2·(y − x²)·(−2x) = −2·(1−x) − 400·x·(y − x²)
f_y = 100·2·(y − x²) = 200·(y − x²)

f_xx = 2 − 400·(y − x²) + 800·x²  = 2 − 400y + 1200x²
f_xy = f_yx = −400x
f_yy = 200

At the minimum (1, 1):
∇f = [ −2·0 − 400·1·0 ]   = [ 0 ]     ← confirmed stationary point
     [       200·0    ]     [ 0 ]

H  = [ 2 − 400·1 + 1200   −400 ]   = [ 802   −400 ]
     [       −400          200 ]     [−400    200 ]

det(H) = 802·200 − (−400)² = 160 400 − 160 000 = 400 > 0
f_xx > 0  (and eigenvalues both positive)
```
→ At `(1, 1)`, `f` has a **local minimum**. The ill-conditioned Hessian (eigenvalues span two orders of magnitude) is exactly why naïve gradient descent struggles on Rosenbrock — motivating Newton's method and its variants (see [`04-optimization.md`](./04-optimization.md)).

## Intuition — Gradient and Hessian: Compass and Curvature Sensor


Partial derivatives describe the landscape **one axis at a time**. But you usually want two bigger things at any point:
1. **Which way is uphill?** → the gradient.
2. **What's the local shape?** (bowl, dome, saddle) → the Hessian.

The gradient packages all first partial derivatives into a single vector. The Hessian packages all second partial derivatives into a matrix.

---

## The gradient — a compass needle on the hill

**The physical image.** Stand anywhere on the topographic map. The **gradient** is a little arrow pinned to your feet, pointing toward the **steepest ascent** from where you are. The *length* of the arrow tells you how steep that climb is.

```
          north ↑
                │                   contours of constant altitude
                │         ╭─────╮
                │      ╭──╯     ╰──╮
                │     ╱   ╭───╮    ╲
                │    │   ╱     ╲    │
                │    │  │  ↗    │   │    ← gradient arrow
                │    │  │ you   │   │      points uphill
                │    │   ╲  │  ╱    │      length = steepness
                │     ╲   ╰─┴─╯    ╱
                │      ╰──────────╯
                │
                └─────────────────────> east
```

### What the gradient tells you

- **Direction** → the fastest-climb direction at this point.
- **Length** → the rate of climb in that fastest direction.
- **Zero gradient** → the arrow has collapsed to a point. You are on a **flat spot** — a peak, a valley floor, or a saddle. These are called *stationary points* (or *critical points*).

### Why it's just the partial derivatives stacked up

Move east a bit → altitude changes by `∂f/∂x`. Move north a bit → by `∂f/∂y`. Combine the two into a vector `(∂f/∂x, ∂f/∂y)` and you get a direction in the plane. That direction happens — by a short geometric argument — to point exactly toward the **steepest climb**.

### What "follow the opposite" gives you

Negating the gradient gives the direction of **steepest descent**. That's exactly what gradient descent uses in optimization and machine learning: stand on the loss landscape, look at the gradient, take a small step in the *opposite* direction. Repeat. You roll downhill.

---

## The Hessian — a curvature sensor

The gradient tells you **where to go next** from here. But once you're at a flat spot (gradient zero), you want to know the **shape** of the region around you — is it a valley, a peak, or a pass?

**The physical image.** A curvature sensor pressed against the ground. It reads three things:

```
   Local shape          Hessian says…          At a flat spot (∇f = 0)…
   ───────────         ─────────────          ─────────────────────────
                                              
        ╲    ╱              curves up          LOCAL MINIMUM
         ╲  ╱               in every           (a cup holding a marble)
          ╲╱                direction
         BOWL               
                             
        ╭──╮              curves down          LOCAL MAXIMUM
       ╱    ╲              in every            (a marble balanced on a dome)
      ╱      ╲             direction            
        DOME                                    
                              
         /\                curves up            SADDLE POINT
        /  \   ↑           in one direction,    (a mountain pass: low along
   ←───X────→             down in another       the ridge, high across)
        \  /   ↓          
         \/                 
       SADDLE              
```

### What the Hessian actually packages

Every second partial derivative, arranged in a matrix:

```
       ┌                ┐
       │  ∂²f/∂x²        ∂²f/∂x∂y  │
 H  =  │                          │
       │  ∂²f/∂y∂x      ∂²f/∂y²   │
       └                ┘
```

Two things are interesting about this matrix:

1. It's **symmetric** (mixed partials commute, for smooth functions) — so it's reasonable to talk about its eigenvalues.
2. Its **eigenvalues are the curvatures along its principal axes** — the cleanest directions in which to read the local shape.

### The curvature test — reading the Hessian

At a flat spot (gradient zero):

| Hessian's eigenvalues | Local shape | What it is |
|---|---|---|
| all positive | bowl in every direction | **local minimum** |
| all negative | dome in every direction | **local maximum** |
| mixed signs | up one way, down another | **saddle point** |
| some zero | flat along some direction | inconclusive — test fails |

The 2D shortcut you'll see in textbooks (`det(H) > 0` with `f_xx > 0` → min, etc.) is just a fast way to check the eigenvalue signs without computing them directly.

## Gradient and Hessian, side by side

| | Gradient `∇f` | Hessian `H` |
|---|---|---|
| Built from | first partial derivatives | second partial derivatives |
| Shape | vector (one component per variable) | square matrix (variables × variables) |
| Physical role | **compass** — points uphill | **curvature sensor** — reads local shape |
| Used when | you want to move (e.g. gradient descent) | you want to classify a stationary point |
| Meaningful value | its **direction + length** | its **eigenvalues** (curvatures) |

---

**What to remember.**

The gradient is a **compass** — points toward steepest ascent, length measures steepness. The Hessian is a **curvature sensor** — its eigenvalues tell you whether the landscape is a bowl (minimum), a dome (maximum), or a saddle. Together they give you a complete local picture: where to step next, and what the terrain around you is shaped like.
