# 3.2 — Derivatives of Common Functions

**What it does:**
A reference table of derivatives for the most common function families. Every complicated derivative reduces to these building blocks combined with the chain/product/quotient rules.

Imagine a **toolbox**. A carpenter does not reinvent the hammer every time — they grab the right tool from the box. Similarly, when differentiating `sin(3x² + 1)`, you grab `(sin u)' = u'·cos u` and `(xⁿ)' = n·xⁿ⁻¹` from the toolbox and combine them. Memorizing this list is the foundation of calculus.

## Intuition — Why Each Tool Has Its Shape

The basic derivatives are usually memorized as a table. That's fine — they are your daily tools. But each one has a physical *reason* for being what it is. Once you see the reason, the entry stops being a random fact and becomes a consequence of what the function actually does.

**The physical image.** A carpenter's toolbox. Each tool has a shape that fits a specific job. You don't reinvent the hammer every day — you grab the right one from the box.

---

### Constants — the tool that does nothing

```
      y
      │
      │─────────── k      ← the curve is a flat line
      │
      └───────────── x
```

A constant function never changes, no matter where you are. Its rate of change is zero, everywhere. Derivative of a constant = **0**. No physical effort needed.

---

### Powers `xⁿ` — stretching layered `n` times

Think of `x²` as "x times x": a **square** of side `x`. If you push `x` up by a tiny amount, the square grows on two sides — one strip of width `x`, twice. So the growth rate is `2x`.

Now think of `x³` as a **cube** of side `x`. A tiny growth of `x` adds three face-slabs, each of area `x²`. Growth rate: `3x²`.

In general, `xⁿ` is a stretching layered `n` times. A tiny growth in `x` multiplies through each of the `n` copies, contributing one term each → **derivative is `n · xⁿ⁻¹`**.

```
      x²                          x³
      ┌──────┬──┐                ┌──────┬──┐
      │      │  │               ╱│      │  │
      │ x·x  │Δx│              ╱ │ x·x  │Δx│
      │      │  │             ╱  │      │  │
      ├──────┼──┤            ╱___├──────┼──┤
      │  Δx  │  │           │ Δx │  Δx  │  │
      └──────┴──┘           └────┴──────┴──┘
      2 strips grow         3 face-slabs grow
      → rate = 2x           → rate = 3x²
```

---

### `eˣ` — the self-replicating function

This one is magic. It is the *unique* function (up to scaling) whose derivative equals itself:

```
          rate of growth = current amount
```

**The physical image.** Compound interest, continuously compounded. Imagine a bank account that pays interest at every instant, proportional to how much money is already in the account. The more you have, the faster you grow. Your growth rate is literally your balance. That is `eˣ`.

Derivative of `eˣ` = **`eˣ`**. Every level of `eˣ` (first, second, hundredth) is still just `eˣ` — it never morphs into anything else.

---

### `ln(x)` — `eˣ` seen from the other side

`ln(x)` is the inverse of `eˣ` — they reflect each other across the `y = x` line. When two functions are reflections of each other, their **slopes at matching points get swapped reciprocally**: if `eˣ` has slope `y` at height `y`, then `ln` at position `y` has slope `1/y`.

That is why derivative of `ln(x)` = **`1/x`**. The `1/x` is just the reciprocal of the original slope, because of the reflection.

```
      y                            y
      │       eˣ                   │   ln(x)
      │      ╱                     │ ╱
      │     ╱                      │╱
      │   ╱                       ╱│
      │  ╱                      ╱  │
      │ ╱                     ╱    │
      └─────────── x         ────────── x
     slope at height y:       slope at position y:
        equals y                  equals 1/y
```

---

### `sin(x)` and `cos(x)` — a quarter-turn dance

The derivative of `sin(x)` is `cos(x)`. The derivative of `cos(x)` is `−sin(x)`. Differentiate again → back to `−sin`, then `−cos`, then `sin`. The pattern loops every four derivatives, each shifting the wave by a **quarter period (90°)**.

```
         sin   ──d/dx──→   cos   ──d/dx──→   −sin   ──d/dx──→   −cos   ──d/dx──→   sin
          ↑                                                                          │
          └──────────────────────── loop of 4 ──────────────────────────────────────┘
```

**The physical image.** A point turning around a circle at constant speed. Its x-coordinate is `cos(t)`, its y-coordinate is `sin(t)`. Its **velocity vector** is tangent to the circle — rotated exactly 90° from the position vector. Differentiating once = rotating 90°. That's why sin and cos shuffle into each other.

---

### `1/x` — the plunger

`1/x` dives toward zero as `x` grows. Its slope is always negative, and steep near `x = 0`, flat far away.

Think of pressing a plunger: small push → big drop in height near the start, then the drop slows down. The rate of drop is sharpest close to the top (`x` small), gentlest far from it. That's why the derivative is `−1/x²` — always negative, and exploding near zero.

---

### `√x` — the slowing climb

`√x` grows fast at first then tapers off — the curve flattens as `x` grows. Its slope starts huge (vertical near `x = 0`) and fades to near-zero for large `x`. Derivative is `1 / (2√x)` — inversely proportional to how high you already are.

```
      y = √x
      │╱
      │─                 slope at x = 0 → infinite
      │ ──                slope at x large → tiny
      │   ────
      │       ─────
      └──────────── x
```

---

### One row recap — why each derivative has the shape it does

| Function | Derivative | Physical reason |
|---|---|---|
| constant `k` | `0` | never changes |
| `xⁿ` | `n · xⁿ⁻¹` | `n` copies, each contributing one layer of growth |
| `eˣ` | `eˣ` | self-replicating — growth rate = current value |
| `ln(x)` | `1/x` | reflection of `eˣ` → slopes get reciprocated |
| `sin(x)` | `cos(x)` | differentiating = rotating 90° on the circle |
| `cos(x)` | `−sin(x)` | same 90° rotation, one step further |
| `1/x` | `−1/x²` | plunger — drops fastest near zero |
| `√x` | `1/(2√x)` | climb slows the higher you go |

**What to remember.** Every entry in the derivative table has a physical shape behind it. `eˣ` is the only one that survives untouched. `sin`/`cos` dance in a 4-cycle of 90° rotations. `ln` is just `eˣ` reflected, with reciprocated slopes. Powers scale as `n · xⁿ⁻¹` because `n` identical copies all contribute when you push `x`. The table is less scary once you see it as a map of physical reasons rather than a pile of facts.

## Core table (basic + composed form with `u = u(x)`)

| Function               | Derivative (simple)       | Derivative with u = u(x)        |
|------------------------|---------------------------|---------------------------------|
| k (constant)           | 0                         | —                               |
| ax + b (linear)        | a                         | —                               |
| x²                     | 2x                        | —                               |
| ax² + bx + c (quadratic)| 2ax + b                  | —                               |
| xⁿ (power)             | n · xⁿ⁻¹                  | n · uⁿ⁻¹ · u'                   |
| 1/x (inverse)          | −1 / x²                   | −u' / u²                        |
| √x                     | 1 / (2√x)                 | u' / (2√u)                      |
| eˣ (exponential)       | eˣ                        | u' · eᵘ                         |
| aˣ                     | aˣ · ln(a)                | u' · aᵘ · ln(a)                 |
| ln(x) (natural log)    | 1 / x                     | u' / u                          |
| log_a(x)               | 1 / (x · ln a)            | u' / (u · ln a)                 |
| sin(x)                 | cos(x)                    | u' · cos(u)                     |
| cos(x)                 | −sin(x)                   | −u' · sin(u)                    |
| tan(x)                 | 1 + tan²(x) = 1 / cos²(x) | u' · (1 + tan²(u))              |

## Combination rules (detailed in 3.3 – 3.6)

```
(k · u)'    = k · u'                            [scalar rule]
(u + v)'    = u' + v'                           [sum rule]
(u · v)'    = u' · v + u · v'                   [product rule]
(u / v)'    = (u' · v − u · v') / v²            [quotient rule, corollary of product rule]
(f ∘ g)'(x) = f'(g(x)) · g'(x)                  [chain rule]
```

## Simple example

`f(x) = 3x² + 2x − 5`
```
f'(x) = 2·3x + 2 = 6x + 2
```

## Complex example (combining rules)

*(a) Product + chain + log:*  `f(x) = ln(x² + 1) · sin(x)`

Let `u = ln(x² + 1)` and `v = sin(x)`. Product rule:
```
f'(x) = u' · v + u · v'

u = ln(x² + 1):   u' = (x² + 1)' / (x² + 1) = 2x / (x² + 1)
v = sin(x):       v' = cos(x)

f'(x) = [2x / (x² + 1)] · sin(x) + ln(x² + 1) · cos(x)
```

*(b) Power + chain:*  `g(x) = (2x + 1)⁵`

Let `u = 2x + 1`, so `u' = 2`:
```
g'(x) = 5 · u⁴ · u' = 5 · (2x + 1)⁴ · 2 = 10 · (2x + 1)⁴
```

*(c) Quotient + trig:*  `h(x) = sin(x) / (x² + 1)`

Let `u = sin(x)`, `v = x² + 1`:
```
u' = cos(x),   v' = 2x

h'(x) = (u'·v − u·v') / v²
      = (cos(x)·(x² + 1) − sin(x)·2x) / (x² + 1)²
```
