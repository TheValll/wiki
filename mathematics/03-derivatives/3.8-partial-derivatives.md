# 3.8 — Partial Derivatives

**What it does:**
Extends the notion of derivative to functions of **several variables**. The partial derivative `∂f/∂x` measures how `f(x, y, ...)` changes when you vary only `x`, holding the other variables fixed. Core tool for gradient descent, backpropagation, physics of fields, and multivariate optimization.

Imagine a **topographic map**: altitude `f(x, y)` depends on east-west (`x`) and north-south (`y`) position. If you stand at one spot and walk **only east** for a tiny step, how fast does your altitude change? That slope is `∂f/∂x`. Walk **only north**? That's `∂f/∂y`. Each partial derivative is a slice of the landscape along one direction — you freeze everything else and do a regular 1D derivative.

**Notation:**
```
Lagrange-style :   f_x,  f_y,  f_xy (mixed)
Leibniz-style  :   ∂f/∂x,  ∂f/∂y,  ∂²f/(∂x∂y)
```

**How to compute `∂f/∂x`:**
Treat **every variable except `x`** as a constant, then differentiate normally with respect to `x`.

**Simple example:**
```
f(x, y) = 3x²y + 4y³ + 5x

∂f/∂x  =  6xy + 0 + 5   (y is frozen, so 4y³ → 0)
       =  6xy + 5

∂f/∂y  =  3x² + 12y² + 0   (x is frozen, so 5x → 0)
       =  3x² + 12y²
```

**Complex example (loss function in ML regression):**
Given prediction `ŷ = w·x + b` and squared loss `L = (y − ŷ)² = (y − wx − b)²`, compute how `L` responds to each parameter:
```
∂L/∂w  =  2·(y − wx − b)·(−x)  =  −2x·(y − ŷ)
∂L/∂b  =  2·(y − wx − b)·(−1)  =  −2·(y − ŷ)
```
→ These are the raw gradients used in **gradient descent**: each weight `w, b` is updated in the direction that reduces the loss. See [`04-optimization.md`](./04-optimization.md) for how this powers training.

## Intuition — Partial Derivatives: One Direction at a Time


When a function depends on several variables, you can't ask "what's the slope?" — there's no single slope. The graph is now a **landscape**, and the slope depends on which direction you step. A partial derivative answers that question along **one axis at a time**.

**The physical image.** A topographic map of a hill. Your altitude `f(x, y)` depends on where you are — east-west (`x`) and north-south (`y`). You stand at one point. Someone asks:

> "If you take a tiny step **east**, how much does your altitude change?"

That rate is `∂f/∂x` — the partial derivative with respect to `x`. Your north-south position `y` is frozen during the step. You're only looking at the east-west slope.

> "What if you step **north** instead?"

That rate is `∂f/∂y` — east-west `x` frozen, north-south slope read off.

---

### The picture

```
          north ↑
                │
                │                   contours of constant altitude
                │         ╭─────╮                (imagine a hill)
                │      ╭──╯     ╰──╮
                │     ╱   ╭───╮    ╲
                │    │   ╱     ╲    │
                │    │  │   ▲   │   │
                │    │   ╲  │  ╱    │
                │     ╲   ╰─┼─╯    ╱    ← you stand here
                │      ╰────┼─────╯
                │           │
                │           │
                └───────────┼─────────> east
                            │
                      step east  → ∂f/∂x  (altitude change per step east)
                      step north → ∂f/∂y  (altitude change per step north)
```

Each partial derivative is a **1D slope along one axis**, measured with all other variables held still.

---

### How to compute `∂f/∂x`

Treat every variable *except* `x` as a constant, then differentiate normally with respect to `x`.

That's it. The rules you already know (sum, product, chain) all apply identically — you just pretend the other variables are fixed numbers for that moment.

---

### Why not a single slope?

Because on a 2D landscape, **different directions have different slopes**. Walking east from a ridge might climb steeply; walking north along the ridge might be flat. The two numbers `∂f/∂x` and `∂f/∂y` capture the slopes of the two standard directions. Other directions are combinations of these two — see the gradient in [`3.9-gradient-hessian.md`](./3.9-gradient-hessian.md).

---

### Generalizes to any number of variables

`f(x₁, x₂, …, xₙ)` gives `n` partial derivatives `∂f/∂x₁, ∂f/∂x₂, …`. For each one, freeze every variable except that one, then differentiate. The concept doesn't change — you're just adding more axes to the landscape.

---

### Where partial derivatives show up

| Domain | Partial derivative role |
|---|---|
| **Gradient descent / backprop** | `∂L/∂w` tells you how the loss changes if you tweak weight `w` alone |
| **Physics of fields** | `∂T/∂x` = how temperature changes per meter in the x-direction |
| **Thermodynamics** | `∂P/∂V` = how pressure changes with volume at fixed temperature |
| **Robotics (Jacobian)** | each entry is `∂(end-effector coord) / ∂(joint angle)` |

---

**What to remember.**

A partial derivative is a **one-axis slope** on a multi-axis landscape. Freeze every variable except the one you're varying, then differentiate normally. You get one number per axis — and together they form the **gradient**, which encodes the full local behavior of the landscape.
