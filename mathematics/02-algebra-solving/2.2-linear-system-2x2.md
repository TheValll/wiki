# 2.2 — Linear Systems (2-Variable Case + General Form)

**What it does:**
Finds the values of the unknowns that satisfy several linear equations simultaneously. The 2-variable case is the geometric entry point (intersection of two lines in the plane); the general `m × n` case is a single matrix equation `A·x = b`.

Imagine two **detectives** arriving at a crime scene, each with a partial testimony. Alone, each testimony describes an infinite number of possible suspects (a whole line of them). But the **one suspect** that matches *both* testimonies is the intersection of the two lines — a single point. Solving the system is cross-checking the two stories to narrow down the unique answer.

## Intuition — Where Do the Lines Cross?

Each linear equation in two unknowns describes a **line** in the plane. Solving two equations at once means: *find the point that sits on both lines at the same time.*

**The physical image.** Think of two roads drawn on a flat map. Any point on road 1 is a potential answer to equation 1. Any point on road 2 is a potential answer to equation 2. The *only* point that satisfies **both** equations is where the two roads cross. That single intersection is the solution `(x, y)`.

The two roads can land in three configurations. The configuration alone tells you how many solutions exist — before any arithmetic.

---

### Case 1 — The roads cross (one solution)

Two lines with different slopes meet at exactly one point.

```
         line 1
            ╲               ╱ line 2
             ╲             ╱
              ╲           ╱
               ╲         ╱
                ╲       ╱
                 ╲     ╱
                  ╲   ╱
                   ╲ ╱
                    •   ← unique (x, y)
                   ╱ ╲
                  ╱   ╲
```

One point satisfies both equations → **unique solution**.

---

### Case 2 — The roads are parallel but distinct (no solution)

Two lines with the same slope, offset from each other. They never meet.

```
     ──── line 1 ─────────────────

     ──── line 2 ─────────────────
```

The two equations demand *different* offsets along the same direction. No point can satisfy both. **No solution.**

---

### Case 3 — The two roads are actually the same road (infinite solutions)

Same slope, same offset — the two lines coincide.

```
     ════════ line 1 = line 2 ════════
```

Every point on the line works. The two equations say the exact same thing. **Infinite solutions.**

---

### Substitution vs Cramer — two paths, one destination

Once you know the lines cross at one point, there are two ways to find that point:

- **Substitution.** Use one equation to express `x` in terms of `y`, plug into the other, solve for `y`, back out `x`. A *sequential* path — one variable at a time.
- **Cramer.** Write everything as a ratio of determinants. A *symmetric* path — both unknowns fall out of two parallel formulas.

They land on the exact same point. Substitution is usually faster by hand; Cramer scales better, extends naturally to larger systems, and makes the role of the coefficient structure more visible.

---

### The three cases in one table

| Configuration of the two lines | Solutions |
|---|---|
| Different slopes → lines cross | **one** unique `(x, y)` |
| Same slope, different offset → parallel | **none** |
| Same line | **infinite** |

**What to remember.** A 2×2 linear system is a **question about two lines on a map**: where do they cross? Most of the time they do, and you get one unique `(x, y)`. The two methods (substitution, Cramer) are two routes to the same intersection — pick whichever feels cleaner for the problem in front of you.

## Formulas — two methods

*Substitution:*
```
From eq₁:  x = (c₁ − b₁·y) / a₁
Substitute into eq₂, solve for y, back-substitute for x.
```

*Cramer's rule:* for
```
a₁·x + b₁·y = c₁
a₂·x + b₂·y = c₂
```
```
D  = | a₁  b₁ | = a₁b₂ − a₂b₁
     | a₂  b₂ |

x = (c₁·b₂ − c₂·b₁) / D
y = (a₁·c₂ − a₂·c₁) / D
```

## Simple example (substitution)

```
2x + y = 7
 x − y = 2

From eq₂:  x = 2 + y
Into eq₁:  2(2 + y) + y = 7   ⇒   4 + 3y = 7   ⇒   y = 1
x = 2 + 1 = 3
```
→ `(x, y) = (3, 1)`.

## Complex example (Cramer's rule)

```
3x + 2y = 16
5x − 4y = 10

D  = 3·(−4) − 5·2   = −22
Dx = 16·(−4) − 10·2 = −84
Dy = 3·10 − 5·16    = −50

x = −84 / −22 = 42/11 ≈ 3.818
y = −50 / −22 = 25/11 ≈ 2.273
```

## Generalization — `m` equations, `n` unknowns

The 2-variable case is the small picture. In practice (linear regression, robotics, simulations) you often face many equations on many unknowns at once. The shape changes, but the logic is the same.

### General form

`m` equations on `n` unknowns `x₁, …, xₙ ∈ ℝ`:

```
  a₁₁·x₁ + a₁₂·x₂ + … + a₁ₙ·xₙ = b₁
  a₂₁·x₁ + a₂₂·x₂ + … + a₂ₙ·xₙ = b₂
                ⋮
  aₘ₁·x₁ + aₘ₂·x₂ + … + aₘₙ·xₙ = bₘ
```

where the **coefficients** `aᵢⱼ ∈ ℝ` and the **right-hand sides** `bᵢ ∈ ℝ` are given. A **solution** is any `n`-tuple `(x₁, …, xₙ) ∈ ℝⁿ` that satisfies every equation simultaneously.

| Symbol | Index range | Role |
|---|---|---|
| `aᵢⱼ` | `i = 1…m`, `j = 1…n` | Coefficient of unknown `xⱼ` in equation `i` |
| `bᵢ` | `i = 1…m` | Right-hand side of equation `i` |
| `xⱼ` | `j = 1…n` | Unknown |

### The three cases — generalized

The three configurations from the 2D picture (cross / parallel / same line) are not a 2-variable quirk — they are the **only** possibilities for any real-valued linear system:

| Solutions | When | Geometric picture |
|---|---|---|
| **None** | Equations contradict each other (e.g. `2x₁+3x₃ = 5` derived from rows 1+2, but row 3 says `2x₁+3x₃ = 1`) | Lines / planes / hyperplanes that miss each other |
| **Exactly one** | Equations pin down a single point | Lines crossing at one point; planes meeting at one corner |
| **Infinitely many** | At least one redundant equation; some unknowns become free parameters | Lines coinciding; planes sharing a whole line; planes coinciding |

> *Linear regression* (MML Chapter 9) deals with the **no-solution** case: when no `x` can satisfy `Ax = b` exactly, find the `x` that makes the residual `‖Ax − b‖` as small as possible.

### Geometric picture beyond 2D

Each linear equation defines a flat object in the unknown space:

| `n` (unknowns) | One equation defines | Solution set is the intersection of `m` such objects |
|---|---|---|
| `n = 2` | a **line** in the plane `(x₁, x₂)` | point / empty / line |
| `n = 3` | a **plane** in 3D space `(x₁, x₂, x₃)` | point / empty / line / plane |
| `n ≥ 4` | a **hyperplane** in `ℝⁿ` | point / empty / lower-dim affine subspace |

The three cases never go away — the geometry just gets harder to draw.

## Compact notation — `Ax = b`

Writing `m` equations row by row is verbose. Stacking the coefficients into a **matrix** and the unknowns / RHS into **column vectors** gives a single compact equation that scales to any size.

### Step 1 — Column form

Group the coefficients of each unknown into a column vector. The system becomes a **linear combination of columns** weighted by the unknowns:

```
   ┌ a₁₁ ┐         ┌ a₁₂ ┐               ┌ a₁ₙ ┐     ┌ b₁ ┐
   │  ⋮  │ · x₁ +  │  ⋮  │ · x₂ +  …  +  │  ⋮  │·xₙ = │ ⋮  │
   └ aₘ₁ ┘         └ aₘ₂ ┘               └ aₘₙ ┘     └ bₘ ┘
```

This is exactly the **"recipe for mixing columns"** picture from [§1.3 Matrix × Vector](../01-linear-algebra/1.3-matrix-vector.md): the unknowns `xⱼ` are the dosages, the columns of `A` are the ingredients, and `b` is the target mixture. *Solving the system = finding the recipe that produces `b`.*

### Step 2 — Matrix form

Pack the columns side by side into the `m × n` matrix `A` and stack the unknowns into the `n × 1` column `x`:

```
        ┌ a₁₁  a₁₂  ⋯  a₁ₙ ┐     ┌ x₁ ┐     ┌ b₁ ┐
   A =  │  ⋮    ⋮       ⋮  │ ,   x = │ ⋮  │,    b = │ ⋮  │
        └ aₘ₁  aₘ₂  ⋯  aₘₙ ┘     └ xₙ ┘     └ bₘ ┘
```

Then the entire system collapses to a single equation:

```
                    A·x = b
```

| Object | Shape | Role |
|---|---|---|
| `A` | `m × n` | Coefficient matrix — one row per equation, one column per unknown |
| `x` | `n × 1` | Vector of unknowns |
| `b` | `m × 1` | Right-hand-side vector |

This compact form is the entry point for everything that follows: [Gaussian elimination](./2.3-gaussian-elimination.md) operates on `[A | b]`, the [matrix inverse](../01-linear-algebra/1.5-inverse.md) gives `x = A⁻¹b` when `A` is square and invertible, and the [determinant](../01-linear-algebra/1.6-determinant.md) tells you whether that inverse exists.

### Worked check — the 2×2 case as a special instance

The system from the simple example above:

```
2x + y = 7
 x − y = 2
```

In `Ax = b` form:

```
   A = [2   1]    x = [x]    b = [7]
       [1  −1]        [y]        [2]
```

Cramer's rule from earlier is *exactly* `det(A) = −3`, `det(Aₓ) = −9`, `det(A_y) = −3`, giving `x = 3, y = 1` — the same answer through the new notation.
