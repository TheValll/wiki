# 2.4 — Remarkable Identities

**What it does:**
A handful of algebraic equalities that appear *everywhere* — expanding and factoring polynomials, simplifying expressions in calculus, deriving physics formulas, manipulating norms and covariances in stats. Memorizing them pays compound interest across every domain.

Imagine having a set of **pre-built Lego blocks**. Every time you see `(a + b)²` you can instantly swap it for `a² + 2ab + b²` without thinking — like clicking in a pre-assembled piece. Without the blocks, you'd rebuild the expansion from scratch every time. With them, your algebraic manipulations flow in seconds.

## Intuition — Pre-Cut Lego Blocks

A handful of algebraic equalities show up so often — in geometry, physics, statistics, derivatives, norms, variances — that memorizing their expanded form saves time *every single day*. They are to algebra what multiplication tables are to arithmetic.

**The physical image.** Pre-cut Lego blocks. When you see `(a + b)²`, you don't reconstruct the expansion from scratch — you swap in `a² + 2ab + b²` the way you click in a pre-assembled piece. Small operation, zero thinking.

---

### `(a + b)²` — a square cut into four tiles

Draw a square of side `a + b`. Split each side into a segment of length `a` and a segment of length `b`. The square breaks into **four rectangular tiles**:

```
      ┌──────────────┬──────────┐
      │              │          │
      │              │          │
      │     a × a    │  a × b   │
      │              │          │
      │              │          │
      ├──────────────┼──────────┤
      │              │          │
      │     a × b    │  b × b   │
      │              │          │
      └──────────────┴──────────┘
         a              b
```

Add their areas: `a·a + a·b + a·b + b·b`. That's where the `2ab` comes from — the **cross tiles count twice**, once along each axis.

`(a − b)²` is the same square logic in reverse: instead of building up, you shave strips off an `a × a` square. The cross tiles still count twice, but now they are *subtracted*, giving `a² − 2ab + b²`.

---

### `(a + b)(a − b)` — a rectangle rearranged into a difference of squares

Take a rectangle of length `a + b` and width `a − b`. Cut a corner off, paste it somewhere else, and the rectangle reshapes into a big `a × a` square with a small `b × b` square missing from it:

```
        ┌─────┬─────┐                 ┌─────────┐
        │     │     │                 │         │
        │ a·a │ a·b │                 │   a·a   │
        │     │     │      ─→         │         │
        ├─────┴─────┤                 ├─────┬───┘
        │ a·b   b·b │                 │ b·b │
        └───────────┘                 └─────┘
                                      (b² cut out)
```

Total area: `a² − b²`. This identity is the engine behind **factoring**: any time you see `a² − b²`, you can instantly rewrite it as `(a + b)(a − b)`.

---

### `(a + b + c)²` — same trick, three-way

A square of side `a + b + c` breaks into **nine tiles**. Three are the "pure" squares (`a²`, `b²`, `c²`). The other six are cross tiles — **three distinct cross types** (`ab`, `bc`, `ca`), each appearing twice (once above, once below the diagonal).

That's why the expansion reads `a² + b² + c² + 2(ab + bc + ca)`.

---

### Why this pattern shows up *everywhere*

Whenever you square a sum — and sums of quantities appear *everywhere* (velocities, signals, random variables, coordinates) — you get pure terms plus cross terms. The pure terms live on the diagonal; the cross terms live off-diagonal and count twice. The exact same structure will reappear in:

| Domain | Squared-sum expansion | What the cross term becomes |
|---|---|---|
| **Statistics** | `Var(X + Y) = Var(X) + Var(Y) + 2·Cov(X,Y)` | **covariance** |
| **Linear algebra** | `‖u + v‖² = ‖u‖² + ‖v‖² + 2·(u · v)` | **dot product** |
| **Physics** | kinetic energy of coupled motions | **interaction energy** |

Every time you see a "squared sum", expect the same pattern: **square of each piece + twice the interaction**.

**What to remember.** Remarkable identities are **square-decomposition recipes**: (sum)² = (pure squares) + (cross terms counted twice). Recognize them on sight. The same structure will resurface in statistics (variance), linear algebra (norms), and physics (energies) — with the cross term always carrying the "interaction" between pieces.

## Core identities

```
(a + b)²       = a² + 2ab + b²
(a − b)²       = a² − 2ab + b²
(a + b)(a − b) = a² − b²

(a + b + c)²   = a² + b² + c² + 2(ab + bc + ca)
```

**Expansion of the 3-term square, step by step:**
```
(a + b + c)² = (a + b + c)·(a + b + c)
             = a·a + a·b + a·c
             + b·a + b·b + b·c
             + c·a + c·b + c·c
             = a² + b² + c² + 2ab + 2bc + 2ca
```
The "mixed" terms each appear twice (ab and ba, bc and cb, ca and ac) — hence the factor of 2.

## Simple example

```
(x + 3)²  = x² + 2·x·3 + 3² = x² + 6x + 9
(x − 5)²  = x² − 2·x·5 + 5² = x² − 10x + 25
(x + y + z)² = x² + y² + z² + 2(xy + yz + zx)
```

## Complex example (variance of a sum of random variables)

In statistics, the variance of `X + Y + Z` for **independent** variables uses the 3-term square identity applied to the expectation operator:
```
Var(X + Y + Z) = E[((X + Y + Z) − E[X + Y + Z])²]
              = E[((X − μₓ) + (Y − μᵧ) + (Z − μ_z))²]
              = E[(X−μₓ)²] + E[(Y−μᵧ)²] + E[(Z−μ_z)²]
              +   2·E[(X−μₓ)(Y−μᵧ)]
              +   2·E[(Y−μᵧ)(Z−μ_z)]
              +   2·E[(X−μₓ)(Z−μ_z)]
              = Var(X) + Var(Y) + Var(Z) + 2·Cov(X,Y) + 2·Cov(Y,Z) + 2·Cov(X,Z)
```
→ If X, Y, Z are independent, all covariances vanish and `Var(X+Y+Z) = Var(X) + Var(Y) + Var(Z)`. This is the backbone of **Kalman filter uncertainty propagation** — the robot combines noisy measurements and its uncertainty grows by summing variances (when errors are independent).
