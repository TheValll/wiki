# 2.3 — Solving a 3-Variable System (Gaussian Elimination + Matrices)

**What it does:**
Finds `(x, y, z)` satisfying three linear equations. Gaussian elimination transforms the augmented matrix into upper-triangular form via row operations, then back-substitutes from the bottom row upward.

Imagine you have a **stack of three tangled ropes** and you want to straighten them out. You pull one rope at a time: fix the first rope's end, then use it to untangle the second, then use both to untangle the third. By the end, each rope is independent. Gaussian elimination is exactly this: use each equation to kill one variable in every equation below, leaving a clean staircase (upper-triangular matrix) that you solve bottom-up.

## Intuition — Building a Staircase

A system of three equations in three unknowns describes **three planes in 3D space**. Each plane is a flat sheet of points that satisfies one equation. Solving the system means: *find the single point that sits on all three planes at once.*

**The physical image.** The corner of a room. The floor and two walls are three flat sheets. They meet at exactly one precise spot — the inside corner of the room. That corner is the solution `(x, y, z)`. Gaussian elimination is the algorithm that locates this corner without drawing anything in 3D.

---

### The key idea — turn the system into a staircase

Solving three tangled equations directly is painful. Instead, you **rewrite** the system step by step — without changing its solution set — until it takes the shape of a staircase.

```
 Before                              After Gaussian elimination
 ┌─────────────────┐                  ┌─────────────────┐
 │ a·x + b·y + c·z │                  │ ▪·x + ▪·y + ▪·z │
 │ d·x + e·y + f·z │      ──→         │       ▪·y + ▪·z │
 │ g·x + h·y + i·z │                  │             ▪·z │
 └─────────────────┘                  └─────────────────┘
   every row mixes                    each row has one
   all three variables                fewer variable than
                                      the row above
```

The shape is called **upper triangular** — below the diagonal, everything is zero. The bottom row contains a single unknown, alone.

---

### The two legal moves

You are allowed two operations that preserve the solution set:

1. **Rescale** a row (multiply it by a nonzero number) — the same equation, just rewritten.
2. **Combine** two rows (add a multiple of one row to another) — if both old equations were true, the new one is automatically true.

That's it. Apply them over and over until the staircase appears.

---

### Climbing back up — bottom to top

Once you have the staircase, the solution falls out almost for free:

```
    ▪·x + ▪·y + ▪·z = ...       ← 3rd: plug in y and z, solve x
          ▪·y + ▪·z = ...       ← 2nd: plug in z, solve y
                ▪·z = ...       ← 1st: solve z directly
                  ↑
          bottom of the staircase
          (only one unknown)
```

Start at the bottom — one unknown, solved in one line. Feed that value up to the row above, which now has only one unknown left. Feed up again, and you're done. This climb back up is called **back-substitution**.

---

### Why it works — the geometric picture

Each row operation is a way of **rewording** the equations. The three planes shift around in space, but they always still meet at the same corner. The staircase form just makes that corner easy to read off:

- the last row pins `z` (one plane parallel to two axes),
- the middle row pins `y` given `z`,
- the top row pins `x` given both.

It's the algebraic equivalent of walking along the corner of the room: first go down the `z` edge to the floor, then slide along the `y` edge to the wall, then along the `x` edge to the corner itself.

---

### What about `n` variables?

The same procedure scales to any size. `n` equations, `n` unknowns → row-reduce to a staircase with `n` steps → back-substitute from the bottom. This is why Gaussian elimination is the **standard** way to solve linear systems at any scale. Libraries like NumPy, Eigen, and every linear-algebra toolchain run variants of it under the hood.

**What to remember.** Gaussian elimination is a **staircase builder**: you use legal row moves to isolate one variable at the bottom of the system, then climb back up to recover the others. The geometry never changes — the planes keep meeting at the same corner. You are just repositioning the equations so the corner becomes visible from the last row.

## Formula (workflow)

```
1. Write the augmented matrix [A | b].
2. Row-reduce to upper triangular using Rᵢ ← Rᵢ − k·Rⱼ.
3. Back-substitute: solve the last row first, feed the result up.
```

## Simple example

```
 x +  y +  z = 6
2x −  y + 3z = 14
 x + 2y −  z = 2
```

Augmented:
```
[1   1   1 |  6]
[2  −1   3 | 14]
[1   2  −1 |  2]
```
Row ops:
```
R₂ ← R₂ − 2R₁:   [1   1   1 |  6]
R₃ ← R₃ −  R₁:   [0  −3   1 |  2]
                  [0   1  −2 | −4]

R₃ ← R₃ + (1/3)R₂:
                  [1    1     1  |   6  ]
                  [0   −3     1  |   2  ]
                  [0    0   −5/3 | −10/3]
```
Back-substitute:
```
−5/3 · z = −10/3   ⇒   z = 2
−3y + z = 2         ⇒   −3y + 2 = 2   ⇒   y = 0
 x + y + z = 6      ⇒    x + 0 + 2 = 6   ⇒   x = 4
```
→ `(x, y, z) = (4, 0, 2)`.

## Complex example (simplified 3-joint robot inverse kinematics)

Find joint positions `(q₁, q₂, q₃)` satisfying three sensor equations:
```
 2q₁ +  q₂ +  q₃ =  8
 4q₁ − 6q₂       = −2
−2q₁ + 7q₂ + 2q₃ =  9
```

Augmented:
```
[ 2   1   1 |  8]
[ 4  −6   0 | −2]
[−2   7   2 |  9]
```
Row ops:
```
R₂ ← R₂ − 2R₁:   [ 2   1   1 |   8]
R₃ ← R₃ +  R₁:   [ 0  −8  −2 | −18]
                  [ 0   8   3 |  17]

R₃ ← R₃ +  R₂:   [ 2   1   1 |   8]
                  [ 0  −8  −2 | −18]
                  [ 0   0   1 |  −1]
```
Back-substitute:
```
q₃ = −1
−8q₂ − 2·(−1) = −18   ⇒   −8q₂ = −20   ⇒   q₂ = 2.5
2q₁ + 2.5 + (−1) = 8  ⇒   2q₁ = 6.5    ⇒   q₁ = 3.25
```
→ `(q₁, q₂, q₃) = (3.25, 2.5, −1)`.
