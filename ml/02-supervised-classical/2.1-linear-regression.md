# 2.1 — Linear Regression

**What it does:** Fits a straight line (or hyperplane) through a cloud of labeled points so future inputs can be predicted by reading off the line. The simplest supervised model and the foundation that every later model either extends, squashes, or replaces.

**Family:** Supervised — Regression (linear, parametric)

**Source:** Deisenroth ch 9 · Hastie & Tibshirani (ESL) ch 3 · Géron ch 4

| Input | Output | Trains via | Cost |
|---|---|---|---|
| Vector `x ∈ ℝᵈ` (continuous or one-hot features) | Scalar `ŷ ∈ ℝ` | OLS closed-form **or** gradient descent | Mean Squared Error |

---

## 2.1.1 — Intuition

**The spring image.** Imagine each data point connected by a vertical spring to a wooden ruler floating above the plane. Each spring pulls the ruler towards its point, the strength growing with the **square** of the stretch. The ruler rotates and slides until total spring energy is minimal. That equilibrium ruler is the regression line.

```
Frame 1 — DATA
                 y │            •
                   │        •
                   │     •
                   │  •          •
                   │•
                   └──────────────────► x

Frame 2 — A LINE WE PICK · each point pulls vertically
                 y │            ↓ •
                   │      ↑ •
                   │ -- - - - - - - - - ← line guess
                   │     • ↓
                   │  • ↑        • ↓
                   └──────────────────► x
                  vertical residual = (yᵢ − ŷᵢ)

Frame 3 — SQUARE THE RESIDUALS · big stretches dominate
                  ┌───────────────────────────────┐
                  │  Σᵢ (yᵢ − ŷᵢ)²   ←  what we minimize
                  └───────────────────────────────┘

Frame 4 — EQUILIBRIUM · line where total spring energy is lowest
                 y │              .•
                   │          .•
                   │       .•
                   │    .• •            line: ŷ = w·x + b
                   │.•
                   │•                   residuals balanced ↑↓
                   └──────────────────► x
```

**What to remember:**
- Squared (not absolute) error → **big misses dominate** the fit. One outlier can swing the line.
- The line passes through the centroid `(x̄, ȳ)` — gravity of the cloud.
- "Fit a line" generalises to "fit a hyperplane" the moment `x` has more than one coordinate.

---

## 2.1.2 — Mechanism

Two paths reach the same optimum. Pick by `d` (number of features) and dataset size.

### Path A — Closed form (OLS, normal equations)

Set every partial derivative to zero, solve the linear system. For the multivariate case with design matrix `X ∈ ℝⁿˣᵈ` (a column of 1s prepended to absorb `b`) and target `y ∈ ℝⁿ`:

```
ŵ = (XᵀX)⁻¹ Xᵀy
```

**Pros:** exact, one shot, deterministic.
**Cons:** the `(XᵀX)⁻¹` inversion is `O(d³)` — painful past `d ≈ 10⁴`. Fails outright if `XᵀX` is singular (perfect multicollinearity). In practice solve via QR or SVD instead of inverting.

### Path B — Gradient descent

```
init  w ← 0,  b ← 0
loop  ŷ ← X·w + b
      r ← y − ŷ
      w ← w + α·(2/n)·Xᵀr
      b ← b + α·(2/n)·Σr
until convergence
```

**Pros:** scales to huge `d` and streaming data, tiny memory footprint, drops straight into SGD / mini-batch variants.
**Cons:** requires learning-rate tuning, no principled stopping rule beyond "loss plateaued".

### When to pick which

| Situation | Path |
|---|---|
| `d` ≤ a few thousand, dense | A (closed form via QR) |
| `d` huge, sparse, or streaming | B (SGD) |
| Want an answer with zero hyperparameters | A |
| Will later swap to neural net | B (same plumbing) |

## 2.1.3 — Math

The full derivation lives in math wiki — this section recaps the load-bearing pieces.

```
Loss (MSE):     L(w, b) = (1/n) · Σᵢ (yᵢ − (w·xᵢ + b))²

Closed form:    ŵ = (XᵀX)⁻¹ Xᵀy

Gradient:       ∂L/∂w = −(2/n) · Σᵢ xᵢ·(yᵢ − ŷᵢ)
                ∂L/∂b = −(2/n) · Σᵢ     (yᵢ − ŷᵢ)
```

**Statistical reading.** Linear regression is the **maximum-likelihood estimator** under the assumption `y = wᵀx + b + ε` with `ε ~ 𝒩(0, σ²)`. Minimising MSE = maximising the log-likelihood of the Gaussian noise model. This is why squared error is "natural" rather than arbitrary.

**With regularisation:**
- L2 penalty `λ‖w‖²` → **Ridge** (closed form: `ŵ = (XᵀX + λI)⁻¹ Xᵀy`, always invertible)
- L1 penalty `λ‖w‖₁` → **Lasso** (no closed form; coordinate descent or proximal methods, drives weights to exactly 0)

## 2.1.4 — Schema

**Data flow (matrix form).**

```
       X  (n × d)         w (d)              ŷ (n)
   ┌──────────────┐      ┌────┐           ┌────┐
   │ x₁¹  x₁² … x₁ᵈ│     │ w₁ │           │ ŷ₁ │
   │ x₂¹  x₂² … x₂ᵈ│  ×  │ w₂ │  + b  =   │ ŷ₂ │
   │  ⋮              │     │ ⋮  │           │ ⋮  │
   │ xₙ¹  xₙ² … xₙᵈ│     │ wₐ │           │ ŷₙ │
   └──────────────┘      └────┘           └────┘
```

**Geometric view (1 feature).**

```
     y │                      .•
       │                  .•
       │               .•          residual ↕
       │     • ─ ─ .•──────● ŷᵢ
       │       .• yᵢ
       │    .•
       │ .•                       ŷ = w·x + b   (best line)
       │•
       └──────────────────────────────► x
                  centroid (x̄, ȳ) lies on the line
```

The fitted line is the **orthogonal projection** of `y` onto the column space of `X` — every other linear combination of features sits further from `y` in Euclidean distance.

## 2.1.5 — Use cases

**✓ Where it shines**
- Continuous target with an approximately linear relationship to inputs
- Interpretability matters: each `wⱼ` reads as "Δy per unit Δxⱼ, holding others fixed"
- **Strong baseline** before any heavier model — if linear regression gets `R² = 0.85`, your fancy gradient boosting beating it by 2 points may not justify the loss of interpretability
- Signal extraction in physics / engineering: spring constants, calibration curves, sensor scaling
- Sample-efficient: `d + 1` data points are enough in principle

**✗ Where it breaks**
- Classification problems (use logistic regression instead)
- Strong non-linearity — the line cannot bend; transform features first or move on
- Heavy outliers — squared loss makes one bad point dominate; use Huber loss or RANSAC
- Multicollinearity — when features are near-duplicates, weights become unstable and uninterpretable
- Heteroscedastic noise — variance changes with `x`; predictions still unbiased but uncertainty estimates lie

**Canonical datasets**
- *California Housing* (sklearn) — predict median house price from neighbourhood features
- *Diabetes* (sklearn) — predict disease progression from 10 baseline features
- *Auto MPG* (UCI) — fuel economy from horsepower / weight / year

## 2.1.6 — Hyperparameters

Vanilla OLS has **none**. The moment a variant is in play, the knobs appear:

| Hyperparameter | Effect | Typical range | Symptom if mistuned |
|---|---|---|---|
| `α` (learning rate, GD only) | Step size of each update | 1e-4 → 1e-1 | Too big: loss diverges. Too small: glacial convergence. |
| `λ` (Ridge / Lasso) | Regularisation strength | 1e-4 → 1e2, log-scaled | Too big: weights crushed to 0, underfit. Too small: same as no regularisation. |
| Polynomial degree | Capacity of polynomial features | 1 → 5 | High degree: train R² → 1, test R² collapses (Runge phenomenon). |
| Feature scaling | Required for GD and any regularised variant | standard or min-max | Without it: dominant-magnitude features eat the gradient. |

## 2.1.7 — Evaluation

**Metrics.**

| Metric | Range | Read as |
|---|---|---|
| **RMSE** | `[0, ∞)` | Typical prediction error in target units |
| **MAE** | `[0, ∞)` | Median-style error, robust to outliers |
| **R²** | `(-∞, 1]` | Fraction of variance explained; `0` = same as predicting the mean, `<0` = worse than the mean |
| **Adjusted R²** | `(-∞, 1]` | R² with a penalty for adding features — use when comparing models of different `d` |

> See also: [`mathematics/05-ml-basics/5.4-rmse.md`](../../mathematics/05-ml-basics/5.4-rmse.md)

**Diagnostic plots — the part most people skip.**

| Plot | Should look like | Failure pattern |
|---|---|---|
| Residuals vs fitted | Cloud with no structure, mean ≈ 0 | Curve / funnel = non-linearity or heteroscedasticity |
| Q-Q plot of residuals | Straight diagonal line | Tails curving off = non-Gaussian noise (CIs unreliable) |
| Residuals vs each feature | Cloud, no slope | Slope = a feature still carries unmodelled signal |
| Cook's distance | All points small | Spike = a single point is bending the fit (high leverage) |

**Failure modes to recognise.**

| Mode | Signature | Fix |
|---|---|---|
| **Heteroscedasticity** | Funnel-shaped residual plot | Weighted least squares, log-transform `y`, robust SEs |
| **Multicollinearity** | Huge VIF (> 10), unstable coefficients | Drop a redundant feature, switch to Ridge |
| **Non-linearity** | Curved residual plot | Add polynomial / interaction features, or change model |
| **Outliers** | Huge Cook's distance on 1-2 points | Investigate (real or data error?), use Huber or RANSAC |

---

## 2.1.8 — Variants

| Variant | Key diff | When to prefer |
|---|---|---|
| **Ridge** | Adds `λ‖w‖²` to loss | Multicollinearity present, want stable weights, all features genuinely matter |
| **Lasso** | Adds `λ‖w‖₁` to loss | Want **feature selection** — drives weights exactly to 0 |
| **ElasticNet** | Mix of L1 + L2 | Mid-ground when correlated feature groups exist |
| **Polynomial regression** | Feeds polynomial features `[x, x², x³, …]` | Mild known non-linearity; still linear in parameters |
| **GLM** (Poisson, Gamma, …) | Different link function + non-Gaussian noise | Counts, rates, strictly positive targets |
| **Bayesian linear regression** | Posterior over `w`, not a point estimate | Need calibrated uncertainty, small data |

## 2.1.9 — Implementation pointer

```python
from sklearn.linear_model import LinearRegression, Ridge, Lasso
from sklearn.metrics import mean_squared_error, r2_score
from sklearn.model_selection import train_test_split
import numpy as np

X, y = ...  # X: (n, d), y: (n,)
X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.2, random_state=0)

model = LinearRegression().fit(X_tr, y_tr)
y_hat = model.predict(X_te)

print(f"weights : {model.coef_}")
print(f"bias    : {model.intercept_:.3f}")
print(f"RMSE    : {np.sqrt(mean_squared_error(y_te, y_hat)):.3f}")
print(f"R²      : {r2_score(y_te, y_hat):.3f}")
```

For Ridge / Lasso swap the constructor and pick `alpha` via `RidgeCV` / `LassoCV` (built-in cross-validation).
